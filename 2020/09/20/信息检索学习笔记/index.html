Week1

基本概念

IR:

​ Information retrieval is a field concerned with the structure, analysis, organization, storage, searching, and retrieval of information.(Salton, 1968)

Collection:

​ Fixed set of documents

Goal:

​ Retrieve documents with information that is relevant to the user’s information need and helps the user complete a task

The classic search model

Precision:

​ Fraction of retrieved docs that are relevant to user’s information need

Recall:

​ Fraction of relevant docs in collection that are retrieved

检索方法：

例：Which plays of Shakespeare contain the words Brutus AND Caesar but NOT Calpurnia?

方法一：直接搜索

​ 这种方法直接全文搜索含有Brutus和Caesar的戏剧，然后去掉没有Calpurnia的那一部

  • 缺点：Slow (for large corpora)；NOT Calpurnia is non-trivial；Other operations (e.g., find the word Romans near countrymen) not feasible；Ranked retrieval (best documents to return)

方法二：Term-Document(关联矩阵)

​ 这种方法以被检索的文本为列首，索引词项为行首，构建一个矩阵，如果索引词在文本中出现，则值为1，否则为0。比如Brutus行的值为110100，Caesar行的值为110111，Calpurnia行的值为10111，求出现过Brutus, Caesar, Calpurnia的文本，对三个值进行AND计算即可：

​ 110100 AND 110111 AND 101111 = 100100，所以第一列和第四列的文本即为所求

  • 缺点：当文本集过大的时候，矩阵会非常稀疏。假设共有100万篇文章，每篇文章有1000词，供需6GB的空间来存储。假设一共有50万个不同词汇，此时的Term-Document矩阵就是50w*100w

  • 解决办法：只存1

方法三：Inverted Index(倒排索引)

​ 对文档进行索引，对于每个项，我们存储那些有这个项的文档的索引，用docID来表示。

​ 注意：需要一个可变大小的倒排表（posting list），因为固定大小的数组会面临溢出的问题

  • 倒排索引流程：

​

  • 索引流程：

    • 先把调整后的token展开成序列；

    • 先根据token本身，再根据文档ID进行排序；

    • 把同一个文档中相同的term进行合并；

    • 拆分成字典和倒排表；

    • 加入文档频率信息；

  • Query

    例：找同时含有Brutus和Caesar的文档，处理方法是在字典中找到Brutus和Caesar的位置，然后找到两者对应的链表，对这两个链表进行合并

    ​ 合并的主要想法是找两个指针从这两个链表的头节点开始向后进行循环，如果两者指针指向的空间的值相同则存储该索引，两个指针都前进一个指针；如果两个指针指向的空间的值不同，就让小的那个指针前进一步，然后进行比较；如此循环，直到一个指针指向了链表的尾巴为止。这种算法的时间复杂度是O（x+y），其中x和y分别表示两条链表的长度。

Phrase Queries

方法一：Biword Indexes（双单词索引）

  • 对文档中出现的连续单词都进行双单词索引，比如出现friends，Romans，Countryman，就有两个双单词：friends Romans以及Romans Countryman。接着对双单词构建倒排表和字典。这样的话，我们就可以解决双单词的词组的查找问题。如果是要查找三单词的话，只需要找到对应的两个双单词的倒排链表，然后再进行AND操作就可以了。
  • 缺点：1.对于两个以上的单词词组可能会搜索到含有对应的双单词，但是实际上这些双单词并不相连的问题。2.双单词会导致字典的规模很大
  • 改进：Parse the indexed and perform paet-of-speech-tagging（POST），去掉冠词介词，减少biwords的组合

方法二：Positional Index（位置索引）

  • 具体方法是在构建倒排表的时候不仅仅记录项（term）出现的文档序号，也记录在对应文档中的位置。

  • 具体算法：上一节的算法的拓展，在比对到文档的序号相同的时候，再进行一次类似的算法，判断两个指针指向的单词位置是否是相邻的。当然基于这个算法我们也可以进一步拓展到临近请求（proximity queries），比如索引A在B的三个单词距离内的文档。
